================================================================================
ARCHITECTURE DECISIONS
================================================================================

1. PostgreSQL + Logical Graph (not Neo4j)
   WHY: Faster dev, simpler LLM integration, fewer failure points

2. REST API (not GraphQL)
   WHY: Simpler, more secure, explicit endpoints

3. Keyword-Based Intent (not ML)
   WHY: Deterministic, no hallucination, instant

4. Multi-Stage Validation Pipeline
   WHY: Defense in depth, clear error messages, auditable

5. Flow Tracing as Differentiator
   WHY: Mirrors real ERP workflows, user-centric, shows domain knowledge

See ARCHITECTURE.md for detailed rationale on each decision.

HOW TO RUN
================================================================================

QUICKEST (One command):

  chmod +x start.sh
  ./start.sh

Then open: http://localhost:5173

MANUAL SETUP:

  Backend:
    cd backend
    python3 -m venv venv && source venv/bin/activate
    pip install -r requirements.txt
    cp .env.example .env
    # Edit .env: add GEMINI_API_KEY
    python ingest.py
    python main.py

  Frontend (another terminal):
    cd frontend
    npm install
    npm run dev

  Open: http://localhost:5173

DOCKER:

  cp backend/.env.example backend/.env
  # Edit backend/.env: add GEMINI_API_KEY
  docker-compose up -d
  # http://localhost:5173 (frontend)
  # http://localhost:8000 (API)
